文字元件
基本定義與應用
文字元件（Text Component）曾被稱為原始 JSON 文字，是基岩版顯示富文字的主要方式。它與 Java 版不同，不具備點擊互動功能，僅能透過格式碼修飾樣式。
應用範圍包括：/tellraw 指令、/titleraw 標題、NPC 命名、書本內容、告示牌文字。
限制：計分板（Score）與實體名稱（Selector）元件僅在聊天室或畫面標題等流動資訊中生效，其他地方不顯示。

根節點結構
根節點可以是「純文字字串」或「JSON 物件」。
若為字串：直接視為純文字內容。
若為物件：必須包含一個名為 rawtext 的 JSON 陣列。
rawtext 陣列規則：必須包含至少一個內容元件，若陣列為空則會執行出錯。

內容元件類型與欄位優先權
一個內容元件內只能定義一種內容類型。若一個物件內同時出現多個必要欄位，解析優先權順序如下：
第一優先：translate (翻譯文字元件)
第二優先：text (純文字元件)
第三優先：score (計分板分數元件)
第四優先：selector (實體名稱元件)
若物件內完全沒有上述必要欄位，則該元件不顯示任何內容。

各類型詳細欄位說明
純文字 (Text)：欄位為 text，值為允許跳脫字元的字串。
翻譯文字 (Translate)：欄位為 translate，值為在地化鍵名。選配欄位 with 可為陣列或物件，用於填充翻譯字串中的預留位置。
計分板分數 (Score)：欄位為 score，子欄位包括 name（分數持有者，支援選擇器或 *）及 objective（計分板目標名稱）。
實體名稱 (Selector)：欄位為 selector，值為目標選擇器或玩家名稱。

解析機制與空值處理
解析順序：按陣列排列順序由左至右解析並組合。子級元件會優先解析完成後再回傳給父級。
無內容 (No Content)：代表解析失敗（例如找不到實體或分數），該元件會被直接忽略。
空內容 (Empty Content)：例如 {"text": ""}，這是一個有效的元件，可用於佔位或避免解析錯誤。
格式碼：格式碼（§）在最終生成純文字顯示時才被解析，因此可以將計分板分數與格式碼組合使用。

跳脫字元與格式化規範
支持的跳脫字元：\n (換行)、\ (反斜線)、" (引號)。特殊字元 \u0000 會強制截斷目前字串。
格式說明符：用於翻譯元件。%s 或 %d 代表變數。
按序匹配：使用 %% 匹配下一項參數。
按位匹配：使用 %0 到 %9 匹配指定位置的參數。

最佳化編寫建議
建議一：避免在內容元件中使用擴展的 rawtext 嵌套，改用翻譯元件或拆分元件。
建議二：一個內容元件僅使用一個必要欄位，不要混用 text 與 translate。
建議三：在基岩版環境中避開 $s 與 $d 等特殊符號，以免觸發未預期的格式化錯誤。
建議四：避免產生「無內容」結果，優先使用「空內容」字串確保指令執行的穩定性。
